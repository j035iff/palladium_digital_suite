# Nightbane Morphus Sub-Forge: Technical Specification

## 1. Overview & Architecture

The Morphus Creation Engine is a highly dynamic, nested Sub-Forge located within **Tab 6 (Traits)** of the master [Character Creation Forge](./forge-character_creation.md) sequence. Built upon the [Universal Forge Navigation Engine](./universal_forge_navigation_engine.md), it is designed to seamlessly handle the complex, deeply recursive, and variable-driven tables of the Nightbane Morphus generation rules without overwhelming the user.

**Parent gate:** The master forge **Roll Pending** tab (Tab 5) must be complete before Morphus work begins — Facade stats are finalized upstream; all Morphus logic lives entirely inside this Sub-Forge.

---

## 2. UI Presentation & View Modes

Because the Morphus tables include dozens of nested sub-categories, **Sub-Forge Tab 2** (The Dynamic Trait Forge) features a persistent view toggle:

**[ Toggle View: Guided Wizard | Expert Mode ]**

The engine defaults to **Guided Wizard** for all new sessions to prevent decision paralysis for new players.

### 2.1 Guided Wizard (Default)

| Aspect | Behavior |
|--------|----------|
| **Layout engine** | Renders required tables as visually distinct, vertically stacked blocks (the **Cascading Block Tree**). |
| **Dynamic generation** | Blocks only spawn when their immediate prerequisite is met. For example, if a prompt asks for a `1D4` roll to determine the number of traits, the subsequent dropdown blocks do not exist in the DOM until that integer is inputted. |
| **Target audience** | New players and those requiring strict, step-by-step validation. |

### 2.2 Expert Mode

| Aspect | Behavior |
|--------|----------|
| **Layout engine** | Splits the viewport into a dual-panel layout. The **left panel** houses a collapsible **Master Index** of all 25+ tables (supporting hierarchical nesting). The **right panel** houses a sticky **Trait Cart** tracking the exact required rolls based on the Archetype selection. |
| **Interactive linking** | The player clicks an empty requirement slot in the Trait Cart, which automatically targets and expands the relevant table in the Left Panel Index. Clicking a trait in the Index assigns it to the Cart. |
| **Target audience** | Veteran players who understand the rulebooks and want to build rapidly without vertical scrolling. |

## 3. The 3-Step Tab Sequence

The Morphus Sub-Forge uses the same Continue / Green / Yellow / Red semantics as the master forge. Sub-Forge tabs are linear; **Continue** validates the active sub-tab without locking upstream master-creation choices.

### Tab 1: The Crossroads & Initialization

- **Engine action:** Initializes the **Morphus Live Ledger**, automatically applying base superhuman attributes (e.g., +10 P.S., P.E., Spd; +6 P.P.; innate Martial Arts; base Horror Factor 6).
- **The selection:** The player is presented with a primary fork in the road:
  - **Path 1 (The Macro Table):** The player selects their specific **Appearance Archetype** from the massive master list (e.g., Lycanthrope, Amalgam, Condemned, Bizarre).
  - **Path 2 (The Personality Crafter):** The player bypasses the macro Appearance Table to hand-craft a form symbolic of their character's personality. They input the physical result of a `1D4+2` roll, which establishes the exact number of direct selections they will make on the Characteristics Table.
- **Completion criteria:** A path is chosen (and integer inputted if Path 2 is active), turning the tab **Green**.

### Tab 2: The Dynamic Trait Forge

- **Engine action:** This single tab dynamically populates its layout, title, and required slots based entirely on what was validated in Sub-Forge Tab 1. It replaces the need for 25 separate tabs.
- **Dynamic states (examples):**
  - **Amalgam:** Spawns three required slots: Animal Form, Plant Life, and Mineral.
  - **Condemned:** Spawns two slots for "Infernal", an input for a `1D4` roll, and then generates that many combination slots requiring selections from specific tables (Animal, Alien, Nightmare, etc.).
  - **Path 2 (`1D4+2`):** Dynamically generates a number of blank Characteristics Table drop-downs matching the integer rolled in Sub-Forge Tab 1.
- **Completion criteria:** Every dynamically generated field within this tab has been filled with a valid selection.

### Tab 3: Morphus Review & Validation (Terminal Gate)

- **Engine action:** Displays the fully compiled **Morphus Live Ledger**. The engine prompts the player to manually roll and input any specific variable dice modifiers generated by their chosen traits (e.g., inputting the result of a `+1D4×10` S.D.C. bonus from a Biomechanical limb).
- **Terminal completion:** Once all manual variable inputs are satisfied, the **Finalize Morphus** button becomes active. Clicking this passes a **Complete** validation state up to the master Character Creation Forge, turning **Tab 6 (Traits) Green**.

## 4. Progressive Slot Resolution & Recursion Engine

The Morphus tables frequently utilize parent tables that act as routing stations (e.g., Animal Form routing to Canine, Feline, etc., or Characteristics multiplying into more rolls). The engine handles this via **Progressive Slot Resolution**.

### 4.1 Parent Table Mutation

When a player selects a parent category (e.g., selecting **Canine** from the Animal Form library), the required slot does **not** lock in.

Instead, the slot mutates from `[ Empty: Animal Form ]` into a more specific requirement: `[ Empty: Canine Trait ]`.

The UI instantly routes the player to the specific sub-table (e.g., Wolf, Fox, Hellhound). The slot only locks and turns **Green** when the terminal sub-trait is selected.

### 4.2 Multiplier Recursion

If a player selects a trait that demands multiple additional rolls (e.g., rolling `99–00%: 4 Characteristics` on the Characteristics table), the engine dynamically appends the exact number of new, empty requirement slots to the active list or Trait Cart.

The tab cannot be completed until the original slot is replaced **and** all newly spawned slots are fully resolved.

## 5. Integration Notes (Master Forge)

| Master tab | Morphus relationship |
|------------|----------------------|
| **Tab 5 — Roll Pending** | Facade / single-form dice only. No Morphus tables or slots. |
| **Tab 6 — Traits** | Hosts this Sub-Forge in full. Morphus vitality dice (H.P., S.D.C.) and trait generation occur here. |
| **Tab 8 — Review & Spawn** | Summary only; Morphus must already be finalized on Tab 6. |

**Pillar alignment:**

- **Mechanical Integrity:** All Morphus percentile and dice modifiers are entered manually (physical dice first); auto-roll is dev-only if present at all.
- **Radical Visibility:** Unresolved slots, empty Trait Cart rows, and blocked sub-tabs are shown with explicit reasons — never hidden.
- **Total GM Agency:** Compiled Morphus ledger values and trait picks are inspectable before **Finalize Morphus**.

### Slot plan kinds (routing engine)

| `kind` | Engine behavior |
|--------|-----------------|
| `required` | One mandatory trait-table resolution (`tableId` → hub or leaf). |
| `choice` | Player picks one option from `options` (OR groups). |
| `repeat` | Same table resolved `count` times (e.g. 2× Infernal). |
| `combination_pool` | Player enters `countRoll` (e.g. `1D4`), then picks that many tables from `pool`. |
| `characteristics_multiplier` | Spawns `count` nested Characteristics rolls; reroll when result &gt; `rerollAbovePercentile`. |

`tableId: "characteristics"` references the characteristics **routing** table (not a trait table). `rerollMultiRollResults: true` on a Characteristics target applies the book rule to ignore 91–00% multiplier rows on nested rolls.

When a content schema changes, update the associated example JSON per project convention.
